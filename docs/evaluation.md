# Evaluation

This document will hold the **evaluation protocol** once modelling begins: splits, metrics, baselines, and business-oriented readouts (for example lift at a fixed intervention budget).

Until a dataset and target definition are locked, keep this file descriptive rather than numeric. Avoid placeholder “accuracy = 99%” style claims.

## What will go here

- Primary and secondary metrics (with justification)
- How churn is defined (label) and the prediction horizon
- Calibration and fairness checks (if applicable to the use case)
- Failure analysis: where the model is systematically wrong
